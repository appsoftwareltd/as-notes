We are using Logseq style dates for journal file names YYYY_MM_DD, but ISO like in the template title YYYY-MM-DD. In the tasks view, ordering will use the page title before the page file name. Tasks due and completed tags use ISO style dates. We need to standardise here. Potentially we may need to give the user options on the journal file name format for compatibility with other PKMS, but visibly and functionally, the extension should behave the same regardless.
