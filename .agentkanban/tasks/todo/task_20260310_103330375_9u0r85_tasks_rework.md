### 1. Task metadata — DB schema changes

**Current `tasks` table:**
```sql
id, source_page_id, line, text, done, line_text
```

**Proposed additions:**
```sql
priority INTEGER NULL,        -- 1, 2, 3 (from #P1 / #P2 / #P3); NULL = unset
waiting INTEGER NOT NULL DEFAULT 0,  -- 1 if #W tag present
due_date TEXT NULL            -- ISO date string YYYY-MM-DD from #D-YYYY-MM-DD; NULL = unset
```

The `text` field will strip the hashtag tokens so only the "clean" task description is stored for display. A `raw_tags` field is _not_ needed — the parsed columns are sufficient.

Schema version bump: `SCHEMA_VERSION` from `1` → `2`.


---

### 2. Task metadata — parsing logic

In `IndexService.indexTasksFromContent()`:

- After extracting `text` from the regex match, run it through a new `parseTaskMeta(text)` helper:
  - Strip `#P1`, `#P2`, `#P3` → `priority` 1/2/3 (first match wins; these are mutually exclusive per task)
  - Strip `#W` → `waiting = 1`
  - Strip `#D-YYYY-MM-DD` → `due_date = 'YYYY-MM-DD'`
  - Return `{ cleanText, priority, waiting, dueDate }`
- Store parsed values in the new columns.

Update `TaskRow` interface to include `priority`, `waiting`, `due_date`.

### 4. TaskPanelProvider — grouping views

Add a `groupBy` mode to `TaskPanelProvider`:
- `'page'` — current behaviour (group by source page, ordered by page title)
- `'priority'` — group by priority level (P1 / P2 / P3 / No Priority), within each group ordered by due date then page
- `'dueDate'` — group by due date bucket: Overdue / Today / This Week / Later / No Date; within bucket ordered by date then page
- `'waiting'` — show only waiting tasks, grouped by page

Root-level tree items in non-page modes are `GroupHeaderItem` (a new discriminant kind).

## Implementation complete (463/463 tests passing)

### Summary of changes

**DB schema (SCHEMA_VERSION 1→2):** `tasks` table gains `priority INTEGER`, `waiting INTEGER`, `due_date TEXT`.

**Task metadata parsing:** New `IndexService.parseTaskMeta(text)` static helper strips `#P1`/`#P2`/`#P3`, `#W`, `#D-YYYY-MM-DD` tags from the start of task text. Tags must appear before any other text. `indexTasksFromContent()` uses this when inserting tasks.

**Sidebar:** `viewsContainers.activitybar` added with `as-notes-sidebar` (icon: existing `images/icon.svg`). `as-notes-tasks` view moved from Explorer → sidebar, changed from tree type to `"webview"`.

**TaskPanelProvider:** Rewritten as `WebviewViewProvider`. Sends all tasks to the webview via `postMessage { type: 'update', tasks }`. Routes `navigateTo` and `toggleTask` messages to extension commands.

**Webview (`src/webview/tasks.ts` + `tasks.css`):** Full browser-side panel with:
- Group by: Page / Priority / Due Date / Waiting
- Filters: Todo only / Waiting only
- Colour-coded badges: P1 (red) / P2 (yellow) / P3 (blue) / Waiting (purple) / due dates (overdue=red, today=orange)
- Tailwind v4 with VS Code CSS variable theming

**Build:** PostCSS/Tailwind CSS step added; esbuild webview bundle (`tasks.ts → dist/webview/tasks.js`).

**extension.ts:** Registered via `registerWebviewViewProvider`; new `as-notes.toggleTaskAtLine` command.
